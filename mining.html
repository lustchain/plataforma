Verified reward rules

Exactly how each 2 LST block reward is distributed.

The reward depends on how many public miners are active. The block producer is never counted again inside the public committee, and official/bootstrap addresses are excluded from the public 30% share.

0 public miners

Official emergency production.

Prod-1/prod-2 keep the chain alive as fallback. The official producer of the block receives the full 2 LST.

1 public miner

The public miner receives 2 LST.

The active public miner produces the block. Because there is no second public participant to form a committee, the same producer receives the full 2 LST.

2 or more public miners

70% producer · 30% public committee.

The block producer receives 1.4 LST. The other active public miners divide 0.6 LST. Official/bootstrap nodes receive none of this public committee share.

Transaction priority fees go to the actual producer of the block. The EIP-1559 base fee follows the network protocol and is not part of the 2 LST block reward.
Automatic TX-FEED · V7A24

Public transactions reach local miners.

The V7A24 transaction feed checks the official primary and backup pending-transaction endpoints every second and injects valid raw transactions into the local miner node. This gives public miners a real chance to include transactions and receive the priority fee when they produce a block.

Windows

Automatic second window.

START opens the miner and LUST TX-FEED V7A24. The feed detects local RPC port 18547 or 8545 and logs to ~/LUST-Miner/logs/txfeed-windows.log.

Ubuntu / Linux

Automatic background feed.

The Ubuntu starter launches the feed in the background. It uses local RPC 18547 and P2P 30313. Feed activity is recorded in ~/LUST-Miner/logs/txfeed-linux.log.

Primary + backup

Resilient public endpoints.

The miner checks rpc.lustchain.org/pending-raw first and automatically tries rpc-backup.lustchain.org/pending-raw when needed.

Network safety

Official producer standby.

V7A24 prioritizes active public miners for normal block production. Official/bootstrap producers remain available only as emergency fallback when no public miner is producing, and they are excluded from the passive 30% public committee share while public miners are active.

Security

Never enter your seed.

The official site only asks MetaMask to sign/send a registration transaction. Never type your seed phrase on any website.

Wallet rule

One machine, one wallet.

For clean testing and production, use one registered wallet per miner machine. Do not run the same wallet on many devices at the same time.

Safe start

No isolated mining.

The miner starts with mining off, validates chain ID, peers and synchronization gap, then restarts in mining mode and launches the automatic TX-FEED.
