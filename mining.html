Windows setup · no confusion

After downloading the ZIP, do this on Windows.

This section is for normal users. The site cannot run a BAT file by itself because Windows blocks websites from executing local programs. The package includes the BAT files; the user only double-clicks them after extracting the ZIP.

1Extract fresh folder

Right-click the downloaded lust-v31h2i-miner-windows-FULL.zip and choose Extract All. Do not reuse old H2E/H2D/V31 folders.

2Open registration helper

Inside the extracted folder, double-click REGISTER-MINER-WINDOWS.bat. It opens the official LUST registration page.

3Connect MetaMask

Click Connect wallet, approve MetaMask, and switch/add LUST Chain. The wallet must be the same public 0x address used by the miner.

4Claim faucet gas

If your wallet has no LST for gas, click Claim faucet. This is only gas for Register and Heartbeat. It is not a mining deposit.

5Register wallet

Click Register wallet and confirm in MetaMask. Register is only once per wallet. If already registered, skip this step.

6Activate / Heartbeat

After Register confirms, click Activate / Heartbeat and confirm in MetaMask. This marks the miner active for public selection.

7Start miner

Go back to the extracted folder and double-click START-LUST-MINER-WINDOWS.bat. Keep the window open.

8Check status

Run STATUS-LUST-MINER-WINDOWS.bat. Wait for Canonical YES, Syncing false, and low/zero gap.

9Confirm on chain

If the miner says it found a block, confirm it on the official RPC/explorer. Do not trust local-only “block found” messages.

Reward rules and equal chance

Exactly how each 2 LST block reward works.

LUST mining is wallet based: 1 active wallet = 1 deterministic chance. More CPU/GPU power does not increase the chance. This remains true with thousands of miners.

0 public miners active

Official fallback keeps the chain alive.

prod-1/prod-2 may produce so the network does not stop. The official producer receives the full 2 LST only while there is no active public miner.

1 public miner active

Public first, official fallback only.

If the public miner produces, it receives 2 LST. If it misses and an official fallback produces, the official producer receives 1.4 LST and the active public miner receives 0.6 LST.

2+ public miners active

70% producer · 30% public committee.

The producer receives 1.4 LST. Other active public miners share 0.6 LST. Official wallets are excluded from passive 30% rewards.

Equal chance: the order is deterministic from chain data and wallet addresses. A stronger computer can help stay online, but it does not buy extra tickets and does not increase lottery weight.

Troubleshooting

Common problems and what they mean.

Use this guide after public mining is active.

Gap is not 0

Miner is still syncing.

Wait. If the gap does not reduce, restart the miner. If still stuck, delete only the local miner data folder on that PC, not the wallet file.

Peers are 0

Network connectivity problem.

Check firewall, VPN, router and internet. Bootnodes must be reachable on P2P.

Register fails

Usually wrong network or no gas.

After public mining is active, switch MetaMask to LUST Chain and claim faucet LST before trying again. Register and Heartbeat need native LST gas.

Official fallback blocks

Check the window.

When public miners are active, official fallback blocks should decrease in the recent window. If public miners are offline, fallback production appears to keep the chain alive.

Already registered wallet

Use Active only.

If this wallet already registered before, do not register again. Start the miner, wait for READY/Canonical YES, then send Active / Heartbeat.

Wrong package

Delete old folders.

If the ZIP SHA does not match the official .sha256 file, delete the ZIP/folder and download the V31H2I full miner ZIP again from this page. Old H2D/H2E binaries are blocked for public mining.
