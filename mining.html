Correct setup order

Follow this exact order to start mining.

Faucet gas comes before registration because Register and Active / Heartbeat are on-chain transactions. Download the official V31 package, register the same wallet used by the miner, send Active / Heartbeat, then start the miner and wait for it to be synced/canonical. If you later change the wallet on the same PC, repeat Register and Active / Heartbeat for the new wallet.

Add LUST Chain

Connect MetaMask and add/switch to LUST Chain, chain ID 6923.

Claim faucet gas

Open the faucet and claim a small amount of native LST. This pays gas for presale, registration and heartbeat.

Download V31

Go to downloads

Start the miner

Run the miner and wait for Gap 0, Peers 1+, Syncing False, Canonical YES and READY.

Register wallet

Use the same wallet shown in the miner. Register wallet is done once per wallet. Wait for the transaction to confirm before sending Heartbeat.

Activate / Heartbeat

After Register is confirmed, send Activate / Heartbeat. This marks that wallet as active for mining.

Watch first blocks

Keep the miner open. If it shows READY and Canonical YES but your wallet does not appear after 10–15 minutes, send one more Activate / Heartbeat.

Important: faucet gas is not a mining deposit. It is only a small native LST amount used to pay transaction gas for Register and Heartbeat.
When do I need Register / Heartbeat?
Register wallet is normally done once for each wallet. Do it again only when using a new wallet. Activate / Heartbeat must be sent after Register is confirmed, and again if you change wallet, restart with a new wallet, stay offline, or the miner is READY/Canonical YES but your wallet does not start appearing in live blocks after 10–15 minutes. Do not spam heartbeat every few seconds.
Simple rule: same wallet = no new Register. New wallet = new Register, then Heartbeat. If the miner already shows READY + Canonical YES, but the wallet still does not produce blocks after a few minutes, send one fresh Heartbeat and keep the miner running.

Run checklist

The miner is ready only when all checks pass.

Before telling a community member that mining is working, confirm these fields in the local miner control center.

Sync

Gap 0 and not syncing.

The local block must match the public block and Syncing must be False.

Peers

Peers 1 or more.

For production, more peers is better. If peers stay 0, restart the miner or check firewall/router.

Canonical safety

Hash check is mandatory.

The current Windows V31 package shows Local hash, Public hash, Compare height and Canonical YES/NO. Gap 0 alone is not enough.

If you change wallet on a PC after mining opens: stop/start the miner with the new wallet, wait for READY + Canonical YES, then Register the new wallet and send a fresh Activate / Heartbeat. The old wallet may still appear in rolling live stats for a short time until the window rolls out.

Reward rules

Exactly how each 2 LST block reward works.

The V31 reward model uses a fixed 2 LST block reward and a 70/30 split when two or more public miners are active.

0 public miners

Emergency official fallback.

Official producer keeps the chain alive and receives the full 2 LST only when no public miner is active.

1 public miner

One public miner receives 2 LST.

If there is no other public committee member, the active public producer receives the full block reward.

2+ public miners

70% producer · 30% public committee.

The producer receives 1.4 LST. The other active public miners share 0.6 LST. Official/prod wallets receive none while public miners are active.

Reward accounting is designed around a fixed 2 LST block reward. When two or more public miners are active, the producer receives 70% and the public committee share receives 30%.

Troubleshooting

Common problems and what they mean.

Use this guide after public mining is active.

Gap is not 0

Miner is still syncing.

Wait. If the gap does not reduce, restart the miner. If still stuck, delete only the local miner data folder on that PC, not the wallet file.

Peers are 0

Network connectivity problem.

Check firewall, VPN, router and internet. Bootnodes must be reachable on P2P.

Register fails

Usually wrong network or no gas.

After public mining is active, switch MetaMask to LUST Chain and claim faucet LST before trying again. Register and Heartbeat need native LST gas.

Official fallback blocks

Check the window.

When public miners are active, official fallback blocks should be 0 in a clean recent window. If public miners are offline, fallback production may appear to keep the chain alive.
