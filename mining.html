How to mine

The clean user flow for Windows and Linux miners.

The public miner must be simple: download, extract, start, wait for READY, register wallet, heartbeat, then leave the miner running.

1

Download the official V7A29 package.

Use only the official package published by LUST. Do not run old V7A25, V7A27 or V7A28 packages after V7A29 activation.

2

Extract to a new folder.

On Windows, extract the ZIP and run START-LUST-MINER-WINDOWS.bat. On Linux, extract the TAR.GZ and run the Linux start script.

3

Wait for READY.

The status must show Running YES, peers 1 or more, Syncing False and Gap 0 before registration or heartbeat.

4

Register the miner wallet.

Connect the same wallet used by the miner, add/switch LUST Chain, then send the V7A29 registration transaction.

5

Send Activate / Heartbeat.

Heartbeat proves the miner is active. If a machine was offline or restarted, send heartbeat again from the same wallet.

6

Leave the miner online.

Rewards come from producing blocks and from the 30% public committee share when two or more public miners are active.

Reward rules

Exactly how each 2 LST block reward works.

The tested V7A29 reward design is deterministic and public-miner first. Official nodes are not allowed to receive passive public rewards while real public miners are active.

0 public miners

Emergency official fallback.

If no public miner is active, the official fallback producers keep the chain alive and the block producer receives the full 2 LST.

1 public miner

The public miner receives 2 LST.

With only one active public miner, there is no separate public committee. The public producer receives the full 2 LST.

2+ public miners

70% producer · 30% committee.

The producer receives 1.4 LST. The other active public miners share 0.6 LST. Official/bootstrap wallets are excluded from this public share.

Transaction priority fees go to the actual block producer. The EIP-1559 base fee follows the network protocol and is not counted inside the 2 LST block reward.

Troubleshooting

Common mining problems and what they mean.

Most support issues should be solved by status checks, heartbeat renewal or a clean resync from snapshot.

Gap above 0

Still syncing.

Wait. Do not register or heartbeat until the miner shows Gap 0, peers and Syncing False.

Gap 0 but no rewards

Check canonical hash.

Restart the miner and compare local/public hash. A future package must show this automatically.

No peers

Network or firewall issue.

Allow outbound internet access and keep the miner package in its own folder. Restart after network comes back.

Wrong network in wallet

Switch to LUST Chain.

Use the Add/Switch LUST button before sending Register or Heartbeat.

No LST for gas

Use faucet.

Claim the small faucet amount for the first transaction, then send register and heartbeat.

Old miner version

Replace with V7A29.

Stop old V7A25/V7A27/V7A28 miners and use a fresh V7A29 folder.

Security

Never enter your seed phrase.

The official site only asks the wallet to send registry transactions. Never type your seed phrase or private key on any website.

Wallet rule

One machine, one wallet.

Use one registered wallet per miner machine for clean accounting and fair reward tracking.

Launch rule

Open in waves, not all at once.

V7A29 is ready for bigger controlled testing, but full public release should wait for miner hash checking and the final announcement.
