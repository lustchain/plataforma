Correct mining order

Do the steps in this order.

This is the safest sequence for new community miners. The faucet comes before registration because Register and Heartbeat are on-chain transactions. If you change the wallet on the same PC, repeat Register and Activate / Heartbeat for the new wallet.

Add LUST Chain

Connect MetaMask and add/switch to LUST Chain, chain ID 6923.

Claim faucet gas

Open the faucet and claim a small amount of native LST. This pays gas for registration and heartbeat.

Open faucet

Download V7A29

Download only the current V7A29 miner package for Windows or Ubuntu from the official download links.

Start the miner

Run the miner and wait for Gap 0, Peers 1+, Syncing False, Canonical YES , Canonical YES and READY.

Register wallet

Use the same wallet shown in the miner. Click Register wallet once, wait for the transaction to confirm, and do not change wallet after this step.

Activate / Heartbeat

After Register is confirmed, click Activate / Heartbeat. This marks that wallet as active for mining.

Watch first blocks

Keep the miner open. If it shows READY and Canonical YES but your wallet does not appear after 10–15 minutes, send one more Activate / Heartbeat.

Important: faucet gas is not a mining deposit. It is only a small native LST amount used to pay transaction gas for Register and Heartbeat.
When do I need Register / Heartbeat?
Register wallet is normally done once for each wallet. Do it again only when using a new wallet. Activate / Heartbeat must be sent after Register is confirmed, and again if you change wallet, restart with a new wallet, stay offline, or the miner is READY/Canonical YES but your wallet does not start appearing in live blocks after a few minutes. Do not spam heartbeat every few seconds.

Run checklist

The miner is ready only when all checks pass.

Before telling a community member that mining is working, confirm these fields in the local miner control center.

Sync

Gap 0 and not syncing.

The local block must match the public block and Syncing must be False.

Peers

Peers 1 or more.

For production, more peers is better. If peers stay 0, restart the miner or check firewall/router.

Canonical safety

Hash check is mandatory.

The current Windows V7A29 package shows Local hash, Public hash, Compare height and Canonical YES/NO. Gap 0 alone is not enough.

If you change wallet on a PC: stop/start the miner with the new wallet, wait for READY + Canonical YES, then Register the new wallet and send a fresh Activate / Heartbeat. The old wallet may stop producing immediately, but the new wallet only joins after its own on-chain activation is confirmed.

Reward rules

Exactly how each 2 LST block reward works.

The V7A29 audit confirmed the total reward and the 70/30 split with three public PCs.

0 public miners

Emergency official fallback.

Official producer keeps the chain alive and receives the full 2 LST only when no public miner is active.

1 public miner

One public miner receives 2 LST.

If there is no other public committee member, the active public producer receives the full block reward.

2+ public miners

70% producer · 30% public committee.

The producer receives 1.4 LST. The other active public miners share 0.6 LST. Official/prod wallets receive none while public miners are active.

Clean audit example: 40 blocks produced by PC1, PC2 and PC3 gave TOTAL_DELTA=80 LST, exactly 40 × 2 LST, with PROD1_BLOCKS=0 and OTHER_BLOCKS=0.

Troubleshooting

Common problems and what they mean.

Use this before opening support tickets or restarting servers.

Gap is not 0

Miner is still syncing.

Wait. If the gap does not reduce, restart the miner. If still stuck, delete only the local miner data folder on that PC, not the wallet file.

Peers are 0

Network connectivity problem.

Check firewall, VPN, router and internet. Bootnodes must be reachable on P2P.

Register fails

Usually wrong network or no gas.

Switch MetaMask to LUST Chain and claim faucet LST before trying again. Register and Heartbeat need native LST gas.

PROD1 blocks appear

Check the window.

After V7A29 and with public miners active, official blocks should be 0 in a clean recent window. If public miners are offline, official fallback may appear to keep the chain alive.
